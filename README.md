# 💰 Сімейний бюджет — PWA

Прогресивний веб-додаток (PWA) для ведення сімейного бюджету на базі Google Sheets. Записуйте витрати, доходи та бізнес-операції з телефону чи комп'ютера, перегляньте аналітику з інтерактивними діаграмами — усі дані зберігаються у вашій власній Google-таблиці.

## Можливості

- **Швидкий запис операцій** — три категорії: Витрати, Доходи, Бізнес
- **Автоматичний курс валют** — через `GOOGLEFINANCE`, фіксується як число на момент операції (не формула, що змінюється з часом)
- **Мультивалютність** — до 5 основних валют + довільна додаткова за потреби
- **Автопідказки опису** — на основі раніше введених значень
- **Аналітика** — місячні, річні та порівняльні (рік до року) діаграми, що будуються миттєво на клієнті з кешованих даних
- **Офлайн-режим** — записи зберігаються локально (IndexedDB) і синхронізуються при відновленні мережі
- **Google OAuth** — вхід лише для дозволених учасників родини, кожен запис підписаний автором
- **Встановлюється як додаток** — через "Додати на головний екран" в Chrome (Android/десктоп)

## Архітектура

| Компонент | Технологія | Розташування |
|---|---|---|
| Клієнт (PWA) | HTML/CSS/JS + Chart.js | GitHub Pages (`index.html`) |
| Сервер / API | Google Apps Script | Прив'язаний до Google Sheets (`Code.gs`) |
| База даних | Google Sheets | Ваш особистий файл |

Дані ніколи не зберігаються на сторонньому сервері — Apps Script виконується у вашому Google-акаунті, а GitHub Pages віддає лише статичний клієнтський код.

## Встановлення

### 1. Підготуйте Google Sheets

Створіть (або використайте наявну) таблицю з такими листами:

- **Витрати**, **Доходи**, **Бізнес** — колонки `A:K` (Бізнес — додатково `L:M`):
  `UUID | Created | Updated | Status | Дата | Категорія | Сума | Опис | Валюта | Курс→USD | =USD(formula)` *(Бізнес: + `Курс2→UAH | =UAH(formula)`)*
- **Settings** — довідники категорій, описів, валют (детальна структура колонок — у коментарях `Code.gs`)
- **Monthly Overview**, **Annual Overview**, **Аналіз** — листи зі зведеннями/формулами (за потреби)

### 2. Підключіть Apps Script

1. У Google Sheets: **Розширення → Apps Script**
2. To be updated, file is not in repo. Видаліть демо-код, вставте вміст файлу `Code.gs` з цього репозиторію
3. У верхній частині файлу вкажіть дозволених користувачів:
   ```javascript
   var ALLOWED_USERS = ['your-email@gmail.com', 'partner-email@gmail.com'];
   var USER_PREFIX   = { 'your-email@gmail.com': 'SL', 'partner-email@gmail.com': 'OX' };
   ```
4. **Розгорнути → Нове розгортання → Веб-додаток**
   - Виконати як: **Я**
   - Доступ: **Будь-хто з Google-акаунтом**
5. Скопіюйте отриманий **URL веб-додатку** — знадобиться на наступному кроці

### 3. Налаштуйте Google Cloud OAuth

1. У [Google Cloud Console](https://console.cloud.google.com/) створіть проєкт (або оберіть наявний)
2. **APIs & Services → Credentials → Create Credentials → OAuth Client ID** (тип: Web application)
3. Додайте ваш GitHub Pages домен у **Authorized JavaScript origins**, напр.:
   `https://your-username.github.io`
4. Скопіюйте **Client ID**

### 4. Налаштуйте клієнт (`index.html`)

Відкрийте `index.html`, знайдіть блок конфігурації на початку `<script>` і заповніть:

```javascript
const SCRIPT_URL       = 'URL з кроку 2';
const GOOGLE_CLIENT_ID = 'Client ID з кроку 3';
```

### 5. Розгорніть на GitHub Pages

1. Створіть репозиторій, завантажте `index.html`, `manifest.json` (і `favicon.ico`, якщо є)
2. **Settings → Pages → Source: Deploy from branch → main / root**
3. Дочекайтесь публікації — додаток буде доступний за `https://your-username.github.io/repo-name/`

### 6. Встановіть як додаток

Відкрийте посилання в Chrome на Android чи десктопі → меню **⋮ → Додати на головний екран / Встановити застосунок**.

## Оновлення

- Зміни в `Code.gs` вимагають **нового розгортання** (Apps Script не оновлює активний веб-додаток автоматично)
- Зміни в `index.html` достатньо запушити в репозиторій — GitHub Pages оновиться сам протягом кількох хвилин

## Безпека

- Email-адреси учасників родини зберігаються **лише в `Code.gs`** (приватний, недоступний ззовні) — клієнтський код на GitHub Pages їх не містить
- Доступ до API перевіряється через Google OAuth-токен на кожен запит
- Дані фізично зберігаються у вашому Google Sheets — жодного стороннього бекенду
